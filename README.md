## Overview

PhytoSentinel-AESTIN predicts **which currently-healthy crop fields will become infected
within the next K steps** (default K=3), given the spatial layout of fields, local
weather, and the current state of the epidemic. Predicting *multiple* steps ahead is
deliberate: over several steps the disease arrives via multi-hop paths that a 1-hop
heuristic cannot see, so the task genuinely rewards graph message passing. The central
idea is **DAGCA** (Dynamic Atmospheric Graph Construction): the message-passing weights
between fields are not fixed — they are a learned, differentiable function of
meteorological edge features (wind speed, humidity, wind-direction alignment, distance),
so the effective graph adapts to the weather.

The repository is fully reproducible: a physics-based SEIR simulator generates the
data, a GNN is trained on a **leakage-safe** prediction task, and an ablation study
isolates the effect of DAGCA.

> **Scope & honesty note.** All results here are on **synthetic** data generated by a
> known dispersal kernel. This is a controlled methodological study and a portfolio /
> workshop-grade artifact — **not** a validated real-world forecasting system. See
> [Limitations](#limitations).

## Main contribution and where it sits in the literature

The contribution is an **application**, framed precisely:

> **DAGCA is the first physics-informed, meteorology-conditioned edge-weighting scheme
> applied to plant-epidemic GNNs**, trained end-to-end with the spread-prediction task,
> together with (a) a Bayesian variant that puts a Beta posterior on each dispersal
> weight and (b) a spectral diagnostic (SENR0) that reads a reproduction-number-style
> quantity off the learned graph.

What this is **not**: DAGCA does not learn graph *structure* (which edges exist) — the
k-NN skeleton is fixed; it learns *weights* on those edges. Edge-feature-conditioned
message weighting is itself standard practice (Edge-Conditioned Convolution,
Simonovsky & Komodakis 2017; MPNN, Gilmer et al. 2017; GATv2, Brody et al. 2021). Our
novelty is the **domain application and the physics-informed feature design**, not a new
graph-learning algorithm. We state it that way deliberately so the claim is defensible.

| Component | What it is | Honest status |
|-----------|-----------|----------------|
| **DAGCA** | Edge weights `w_ij = σ(MLP(wind_speed, humidity, wind_alignment, distance))`, used as the *only* message gate | Application-level novelty; mechanism is standard edge-conditioned weighting |
| **Bayesian DAGCA** | Each weight is `Beta(α_ij, β_ij)`; single-sample reparameterized training + KL-to-uniform prior | Provides uncertainty; calibration is *measured* (ECE + reliability diagram), not assumed |
| **PhytoGNN** | GraphSAGE / GATv2 / GCN backbone, message-passing gated by DAGCA weights | Standard, clean implementation |
| **SENR0** | Spectral radius of the learned adjacency via the Next-Generation Matrix | **Diagnostic readout, not a validated epidemiological R₀** (see below) |

### On SENR0 and R₀

SENR0 computes `R₀ = ρ((β/γ)·A)` where `A` is the learned dispersal adjacency, using a
differentiable power iteration. **Caveat we state up front:** `A` is trained against a
classification loss, not fit to recover transmission rates, so `ρ(A)` is a
**graph-connectivity diagnostic** inspired by the NGM formalism — *not* a calibrated
basic reproduction number. We report it as an interpretability signal (and a
mathematically correct one), and we do not claim it equals the true `R₀` of the
underlying process unless the learned weights are separately shown to track transmission.

## The prediction task (leakage-safe by construction)

The SEIR process is monotone (S → E → I → R, no recovery to S). Two leakage traps exist;
both are closed in [`data/synthetic_epidemic.py`](data/synthetic_epidemic.py):

1. **Feature timing.** Node features encode the SEIR state at the *sampled* time `t`
   (one-hot) plus static features — **never** the final state of the simulation.
2. **Evaluation frontier.** A node already E/I/R at `t` is trivially infected at `t+1`.
   We therefore compute the loss and **all metrics only over nodes that are Susceptible
   at `t`** (`eval_mask`). For those nodes the SEIR one-hot is the constant `[1,0,0,0]`,
   so it carries no label information — the model must infer next-step infection from
   position, humidity, and the states of neighbouring nodes via message passing.

This turns the problem from a near-trivial "copy the current state" task into a genuine
spatial spread-prediction task. The positive class (newly infected) is naturally
imbalanced (~10%), so training uses inverse-frequency class weighting and we report
**AUPRC** and **ECE** alongside F1/AUROC.

### What makes the study defensible (not just "synthetic")

- **Multi-step task by design** — predicting K=3 steps ahead forces multi-hop reasoning,
  the regime where message-passing GNNs beat 1-hop heuristics (a single-step version is
  dominated by immediate-neighbour features, where simple baselines win — we verified this).
- **Leakage-safe task** — scored only on currently-susceptible fields, features at time *t*.
- **Isolated ablation** — DAGCA is the sole edge gate, so "DAGCA on/off" measures DAGCA.
- **External baselines** — logistic regression, random forest, tabular MLP, and a
  non-learned infected-neighbour heuristic, on the identical frontier.
- **Cross-physics generalization** — two structurally different ground-truth dispersal
  kernels (cosine alignment vs. Gaussian plume); we train on one and test on the other.
  This is the direct, honest answer to "synthetic results are circular."
- **Validated uncertainty** — temperature scaling + an uncertainty-vs-error analysis,
  not an unproven "calibrated" claim.
- **Multi-seed, shared-data** — every config trains on identical graphs; results
  aggregated over seeds.

## Results

Real results from a 3-seed `experiments/run_study.py` run (cosine physics, base rate ~20%):

| Configuration | DAGCA | F1 | AUROC | AUPRC |
|---------------|-------|----|-------|-------|
| **DetDAGCA + SAGE** | Yes | 0.557 +/- 0.025 | **0.833 +/- 0.017** | **0.585 +/- 0.028** |
| BayesianDAGCA + SAGE | Yes | 0.548 +/- 0.018 | 0.828 +/- 0.016 | 0.571 +/- 0.021 |
| BayesianDAGCA + GAT | Yes | 0.551 +/- 0.013 | 0.824 +/- 0.013 | 0.568 +/- 0.014 |
| No-DAGCA + SAGE | No | 0.529 +/- 0.012 | 0.811 +/- 0.011 | 0.547 +/- 0.017 |
| No-DAGCA + GCN | No | 0.526 +/- 0.018 | 0.810 +/- 0.012 | 0.548 +/- 0.007 |

**DAGCA effect (paired):** dAUPRC +0.038 +/- 0.025, dAUROC +0.022 +/- 0.009 - positive in **3/3 seeds**.

**Cross-physics generalization:** train on cosine, test on plume (OOD) -> **AUROC 0.926, AUPRC 0.800**.

**Uncertainty:** error-correlation +0.997; well-calibrated (T~1.16); selective prediction
lifts accuracy 0.679 -> 0.723 at 80% coverage.

Honest note: the GNN is *competitive with* strong tabular baselines on the in-distribution
task (leads AUROC, ties AUPRC); its distinctive value is generalization + the DAGCA effect
+ usable uncertainty. DAGCA is the only edge gate, so the DAGCA-vs-No-DAGCA comparison
cleanly isolates its contribution.

## Method details

**DAGCA.** Given the k-NN skeleton and edge features `e_ij ∈ ℝ⁴`:
`w_ij = σ(MLP(e_ij · exp(s)))`, with `s` a learned log-scale. `w_ij` is the sole weight
on messages `j → i`.

**Bayesian DAGCA.** `w_ij ~ Beta(α_ij, β_ij)`, `α,β = softplus(BetaHead(e_ij)) + 1`.
Training draws a **single** reparameterized Kumaraswamy sample per forward pass (so
sample-to-sample variance actually reaches the loss; averaging many draws would collapse
it). Loss `= L_task + λ·KL[Beta(α,β) ‖ Beta(1,1)]`. Note `Kumaraswamy(1,1) = Beta(1,1)`,
so sampler and prior coincide exactly at initialization.

**SENR0.** `K = (β/γ)·A`, `R₀ = ρ(K)` via differentiable power iteration — reported as a
diagnostic (see caveat above).

## Limitations

- **Synthetic data only.** Data comes from a known dispersal kernel; the model is
  evaluated on the same generative family it is designed for. Results demonstrate the
  method works *in principle*; they do **not** establish real-world generalization. The
  highest-value next step is one real spatial plant-disease dataset.
- **SENR0 is a diagnostic, not a validated R₀** (see above).
- **No external/domain baselines yet** (e.g., logistic regression on met features,
  classical SEIR fit). The current ablation isolates DAGCA but does not benchmark against
  non-GNN practice.
- **Bayesian uncertainty is reported, not yet validated as decision-useful.** We provide
  ECE and a reliability diagram; we do not yet show high-uncertainty edges correspond to
  genuinely unpredictable dispersal.
